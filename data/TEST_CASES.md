## 数据设计说明

### sample_sales.csv（640行）

| 维度 | 内容 |
|------|------|
| 区域 | 华东、华南、华北、华西、华中 |

### sample_parking_ops.csv（150行）

| 车场 | 设计特征 |
|------|---------|
| A停车场 | 健康稳定，日均收入约10,500 |
| B停车场 | 3月1日起设备故障：收入骤降23%，支付失败率从2%飙升至9.6%，异常开闸日均30+次 |
| C停车场 | 最优表现，微幅增长 |
| D停车场 | 新开业爬坡，首日6,353→末日12,935（翻倍） |
| E停车场 | 社区小车场，收入低（~4,200），包月占比高，免费放行偏多 |

## 一、销售数据查询（source: sales）

### T01 区域趋势对比
```
最近30天华东和华南的成交额趋势
```
**预期结果**：折线图显示两条曲线；华南后半段明显下滑；Claude 应指出华南后15天趋势转弱，建议追问原因

---

### T02 全区域排名汇总
```
最近30天各区域成交额排名
```
**预期结果**：华东 > 华南 > 华北 > 华中 > 华西（大致比例 3:2.2:1.9:1.3:1）

---

### T03 华北增长发现
```
最近30天华北销售增长情况如何？
```
**预期结果**：识别华北30天线性增长约35%，Claude 主动提示这是表现最佳区域

---

### T04 产品排名
```
最近30天哪个产品卖得最好？
```
**预期结果**：A产品 > B产品 > C产品 > D产品；Claude 建议追问各区域产品偏好差异

---

### T05 华南下滑诊断（追问测试）
```
第一轮：最近30天华东和华南成交额趋势
第二轮：为什么华南后半段下滑了？
```
**预期结果**：Claude 结合前一轮数据，分析华南第15天起的下滑，建议查看具体产品或促销活动变化

---

### T06 周末 vs 工作日
```
最近30天周末和工作日的成交额有差异吗？
```
**预期结果**：周末销售高约15%；数据支撑明显

---

### T07 退款率异常
```
最近30天有退款记录吗？哪个区域退款最多？
```
**预期结果**：各区域均有少量退款记录（~8%订单含退款）

---

### T08 时间范围边界
```
上个月（2月份）各区域销售汇总
```
**预期结果**：数据从2月7日起，Claude 应说明数据从2月7日开始，非完整2月

## 二、停车场数据查询（source: parking_ops）

### T09 车场收入排名
```
最近7天各车场收入排名
```
**预期结果**：C停车场最高 > A停车场 > D停车场 > B停车场(骤降) > E停车场最低

---

### T10 B车场异常诊断（核心场景）
```
最近7天哪个车场收入下滑最明显？
```
**预期结果**：明确指出 B停车场，收入从2月均值~9,127跌至3月均值~7,040（-23%），支付失败率异常高（9.6%），Claude 应主动推断设备故障可能

---

### T11 支付失败率专项分析
```
哪个车场支付失败率最高？最近有没有异常？
```
**预期结果**：B停车场支付失败率9.6%，远超其他车场（~1.5-2.5%），属于严重异常

---

### T12 异常开闸分析
```
最近7天各车场异常开闸情况
```
**预期结果**：B停车场异常开闸次数最多，3月8日约为3月1日的3倍以上，呈恶化趋势

---

### T13 D车场增长分析
```
D停车场最近30天运营情况如何？
```
**预期结果**：收入从6,353增长至12,935，增长约104%；车位利用率从50%提升至接近85%；Claude 应判断为新开业爬坡期

---

### T14 E车场盈利结构
```
E停车场的包月和临停收入比例如何？
```
**预期结果**：包月占比约65-70%（高于其他车场），临停占比低，符合社区车场特征

---

### T15 全车场对比报告
```
给我一份最近7天各车场的整体经营报告
```
**预期结果**：触发 `parking_management_report` intent；C最优、D增长最快、B急需整改、E规模小、A稳健

## 三、多轮追问测试（验证 session 上下文）

### T16 连续深挖
```
第1轮：最近30天华东和华南的成交额趋势
第2轮：华南下滑是从哪天开始的？
第3轮：下滑期间A产品和B产品分别是什么情况？
```
**验证点**：每轮都能正确引用上轮结论，不需要重新解释背景

---

### T17 跨数据源追问
```
第1轮：最近7天销售额最高的是哪个区域？
第2轮：那个区域对应的停车场运营情况怎么样？
```
**验证点**：Claude 能识别这是跨数据源问题，分别调用两次 query_data

---

### T18 澄清问题触发
```
销售怎么样？
```
**预期结果**：Claude 识别问题不明确，询问：时间范围是？关注哪个区域/产品？
